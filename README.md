# Solar-farm-inspection
Autonomous inspection of a simulated solar power plant by a Clover quadrocopter in the Gazebo simulator: navigation by aruco tags, detection of overheating and panel contamination, light indication, and automatic reporting.

![](https://github.com/Datamol94/Solar-farm-inspection/edit/main/screenshots/Screenshot_2026-07-03_17-29-09.png?raw=true)

Автономная инспекция панелей на базе Clover, ROS и Gazebo
Отборочный этап соревнования «Архипелаг 2026»

1. Описание проекта
Квадрокоптер Clover в симуляторе Gazebo выполняет автономный облёт поля с aruco-метками, находит солнечные панели по цвету, определяет цвет индикационной площадки рядом с панелью (жёлтый, оранжевый или красный), подсчитывает количество объектов загрязнения на поверхности панели, включает соответствующую световую индикацию и формирует текстовый отчёт по каждой панели.
Пример результата работы скрипта:

Солнечная панель № 1: 5.15 1.00, нормальное состояние, 2
Солнечная панель № 2: 0.86 2.06, срочный ремонт, 4
Солнечная панель № 3: 3.13 3.00, некритический перегрев, 3
Солнечная панель № 4: 7.88 6.03, некритический перегрев, 3
Солнечная панель № 5: 3.19 7.01, нормальное состояние, 4

2. Структура репозитория
Папка launch содержит изменённые файлы clover.launch и aruco.launch.
Папка scripts содержит основной скрипт миссии clover_flight_final.py.
Папка Panels содержит необработанные снимки панелей, создаётся автоматически при полёте.
Папка Annotated содержит снимки с обведёнными зонами загрязнения, создаётся автоматически.
Папка Logs содержит итоговый отчёт panels_report.txt, создаётся автоматически.
В корне репозитория находятся файлы requirements.txt и README.

3. Требования к системе
Для работы требуется Ubuntu 20.04 внутри виртуальной машины с образом Clover и Gazebo, ROS Noetic, пакет Clover с плагинами aruco_pose и clover_blocks, Python 3.8 или новее. На хост-машине должна быть установлена Windows 10 или 11, объём оперативной памяти не менее 8 ГБ, процессор не хуже i5-7400, разрешение экрана не менее 1920 на 1080, скорость интернета не менее 10 Мбит в секунду.

4. Установка окружения
Сначала нужно клонировать репозиторий командой git clone и указать ссылку на репозиторий, затем перейти в папку solar.
Далее нужно установить VMware Workstation Player и импортировать образ операционной системы с симулятором Gazebo и предустановленным Clover, предоставленный организаторами. После запуска виртуальной машины и загрузки рабочего стола нужно установить дополнительные python-зависимости командой pip3 install -r requirements.txt. Пакеты rospy, cv_bridge, sensor_msgs и clover устанавливать отдельно не нужно, они входят в состав ROS и образа Clover.

5. Настройка launch-файлов
Файлы clover.launch и aruco.launch из репозитория нужно скопировать в каталог Clover, заменив исходные файлы по пути ~/catkin_ws/src/clover/clover/launch/.
В файле aruco.launch включены aruco_detect, aruco_map и vpe_publisher, длина маркера установлена равной 0.33 метра, карта загружается из файла map.txt.
В файле clover.launch включена аруко-навигация, включена светодиодная лента, включена основная камера, параметр simulator установлен в значение true, поскольку работа ведётся в симуляторе, а не на реальном дроне.
После копирования файлов нужно пересобрать рабочее пространство командами catkin_make и source devel/setup.bash.

6. Построение карты aruco-меток
Карта поля из 49 меток строится командой rosrun aruco_pose genmap.py 0.335 10 10 1 1 0, результат перенаправляется в файл map.txt по пути ~/catkin_ws/src/clover/aruco_pose/map/map.txt, с флагом --top-left, обозначающим точку отсчёта карты.
В этой команде размер стороны маркера равен 0.335 метра, количество маркеров по осям X и Y равно 10 и 10, расстояние между центрами маркеров по обеим осям равно 1 метру, нумерация начинается с маркера номер 0.

7. Расстановка объектов в Gazebo
Объекты в мире расставлены вручную. Пять моделей солнечных панелей размещены в пределах aruco-карты на расстоянии не менее 2 метров друг от друга. У каждой панели расположена индикационная площадка одного из трёх цветов на расстоянии не более 0.5 метра от панели. На поверхности каждой панели размещено от двух до пяти зелёных прямоугольников, имитирующих загрязнение.
Модели объектов взяты по ссылке, предоставленной организаторами: https://disk.yandex.ru/d/OCIngJKBgHSY5Q

8. Запуск миссии
В первом терминале запускается симулятор и все ноды Clover командой roslaunch с указанием файла запуска мира, использованного при сохранении расстановки объектов.
Во втором терминале запускается основной скрипт миссии командой python3 clover_flight_final.py из папки scripts.
Скрипт выполняет автономный взлёт, облёт маршрута из точек по сетке над aruco-картой, при обнаружении панели зависает над ней, определяет цвет индикационной площадки, включает соответствующую подсветку, сохраняет фотографию панели и фотографию с обведёнными объектами загрязнения, записывает строку в отчёт, после облёта всех точек маршрута возвращается в точку старта и совершает посадку.
